# Stage 0 · PDF 页面去重与空白过滤 · 设计方案

> 目标：在送 MathPix 之前，把不必要的页面过滤掉，节省计费页数，且**全程无需人工确认**。

---

## 一、问题背景

MathPix 按页计费（~$0.004/页），教材中有大量不含数学内容的页面：

| 页面类型 | 举例 | 是否有价值 |
| --- | --- | --- |
| 共用封面/封底 | 出版信息、广告页 | ❌ 跳过 |
| 字导笔记页 | 全页横线，仅标题"字导笔记" | ❌ 跳过 |
| 整页空白 | 完全空白（排版间隔页） | ❌ 跳过 |
| 纯虚线答题框 | 仅有虚线矩形，无题目文字 | ❌ 跳过 |
| 练习题页（含答题框） | 题目文字 + 空白答题框 | ✅ 处理 |
| 例题讲解页 | 完整数学内容 | ✅ 处理 |

**关键约束**：宁可漏掉空白页（多花钱），**绝不误跳内容页（丢数据）**。

---

## 二、四层过滤流水线

```text
所有 PDF 页面
      │
      ▼ L1 规则跳过（最快，O(1)）
      │  前 N 页 + 后 M 页（同套教材共用封面/封底）
      │  default: N=4, M=2，可按 material_set 配置
      │
      ▼ L2 MD5 精确去重（快，跨文件）
      │  渲染 36dpi 灰度缩略图 → MD5 精确匹配
      │  相同 MD5 → 判定为重复，仅保留首次出现的页
      │
      ▼ L3 像素空白检测（可靠，保守阈值）
      │  三个信号必须同时满足才判定为空白：
      │    亮像素比  > 99.5%  （近白 pixel > 240）
      │    暗像素比  < 0.1%   （墨迹 pixel < 80）
      │    边缘密度  < 0.2%   （PIL FIND_EDGES 过滤后）
      │
      ▼ L4 语义内容检测（精确，分 PDF 类型）
      │  text PDF：PyMuPDF get_text() 直接提取 → 正则匹配数学特征词
      │  image PDF：像素密度兜底（bright>0.98 且 dark<0.002 → 跳过）
      │
      ▼
  pages_manifest.jsonl（只含待处理页）
```

---

## 三、判决逻辑细节

### L3 像素阈值选取依据

```text
一张有内容的练习页，哪怕只有一行"练习 10"，
暗像素比通常 1~3%，边缘密度 0.5~2%
远超 0.1% 和 0.2% 的阈值 → 三道防线同时误判概率极低
```

### L4 数学特征词正则

```python
MATH_PATTERNS = [
    r'\d{2,}',                               # 多位数字
    r'[×÷·=≠≈≤≥∑∫√%‰]',                     # 数学符号
    r'[=＝]\s*\d',                           # 等式
    r'[+\-*/]\s*\d',                         # 运算
    r'\b(计算|求|证明|解|设|若|答)\s*[:：]?', # 题目动词
    r'(练习|例题|例\s*\d|习题|第\s*\d+\s*题)', # 题目标记
    r'[(（]\s*\d+\s*[）)]',                   # 编号子题
    r'[A-Za-z]\s*[=＝]\s*[\d(（]',           # 变量赋值
]
```

有任意一条匹配 → PROCESS；全部不匹配且有效字符 < 30 → SKIP。

### 误判方向

```text
                实际有内容    实际无内容
判为 PROCESS   ✅ 正确        ✅ 多花 ¥0.028（可接受）
判为 SKIP      ❌ 丢数据！   ✅ 节省成本

因此：L4 阈值设计为「必须两个条件同时满足才 SKIP」
  条件A：无任何数学特征词匹配
  条件B：有效文字总长 < 30 字符
```

## 六、实测结果（3份 PDF，100 页）

> 测试文件：学而思秘籍 12 级 · 第1/15/16讲（3份，32-36页/份）

### 关键发现

**这批"text_pdf"实质是图片嵌入型 PDF**：每页含一张 4422×6047 全幅图片，无内嵌文字（get_text() 全部返回空）。

**彩色文字的像素特征**：
教材使用橙色/蓝色文字和装饰，转灰度后落在 141~200（中间亮度段），**不是"暗像素"（< 80）**。
因此 dark_ratio = 0.000 的页面实际仍有题目内容，不能按 dark 来判断有无内容。

| 过滤层 | 触发页数 | 原因 |
| --- | --- | --- |
| L1 规则（前4+后2） | 18 页（18%） | 正常触发 |
| L2 MD5 哈希去重 | 0 页 | 3 份不同章节封面各不同，无跨文件真正相同页 |
| L3 像素空白 | 0 页 | 无页面达到 bright>0.995（彩色装饰拉低亮度） |
| L4 语义过滤 | 0 页 | 无页面 bright>0.98（阈值安全，零误报） |
| **合计节省** | **18 页（18%）** | **零误报，零内容丢失** |

### 代码修正记录

**问题**：初版使用 8×8 aHash + 汉明距离≤3，将相似布局但不同内容的页面误判为重复（同文件 23 页被错误标记）。

**根因**：8×8 分辨率只能区分整体亮度分布，结构相似的答题框页哈希值几乎相同。

**修正**：改用 MD5 精确匹配（same vector content → same pixels），消除全部误报。

### 跨文件去重的实际效果

对同一套教材的不同章节（如秘籍 12 级第1/15/16讲），封面图片各异（不同章节标题），**不存在跨文件真正相同页**。
L2 哈希去重的价值体现在：**同一套 PDF 的多份重复拷贝**或**教材内有完全相同的模板页**时（如某些教材的答题纸确实完全空白）。

### 可优化的后续方向

要识别"字导笔记"类的空白填写页，需要对图片 PDF 做 OCR（如 PaddleOCR），对彩色文字才能正确判断是否有数学内容。当前代码在无 OCR 情况下**保守处理（全部送 MathPix）**，无数据丢失风险。
