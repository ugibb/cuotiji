## 最终技术方案（待确认）

🤖 **AI**

### 架构总览

```
学生端（小程序）
    │  发文字 / 图片 / 语音
    ▼
服务端 POST /api/problems/:id/dialogue
    │
    ├─ 读取当前 reviewStage（状态机）
    ├─ 统计本阶段 turnCount
    ├─ 路由到对应 AI 模型（domestic → Qwen2.5-VL, international → Claude Sonnet 4.6）
    ├─ 构造带阶段 System Prompt 的多模态上下文
    ├─ 解析 AI 返回的 JSON（reply + stageComplete + suggestNext）
    └─ 更新 reviewStage / reviewStatus，返回给前端
```

### 实施步骤（共 6 步）

1. **Prisma 迁移**：`Stu_Dialogue` 加 `imageUrl`，确认 `ReviewStage` enum 值
2. **腾讯云 COS 集成**：新增 `POST /api/problems/:id/chat-image` 图片上传端点
3. **AI Service 重构**：多模型路由（Qwen2.5-VL / Claude）+ 状态机 System Prompt + 返回 JSON
4. **对话路由更新**：状态机流转逻辑（turnCount 计数 + 阶段推进 + suggestNext 判断）
5. **小程序图片发送**：启用相机按钮、图片气泡渲染
6. **小程序 suggestNext 联动**：收到信号后显示「进入下一题」卡片

### 状态机各路径

| result | 阶段顺序（每阶段最多 5 轮） |
|--------|--------------------------|
| correct | PROBE_THINKING → VERIFY_DEPTH → EXPLORE_VARIANTS → COMPLETE |
| wrong | PROBE_THINKING → IDENTIFY_ERROR → ROOT_CAUSE → COMPLETE |
| unknown | GUIDE_READING → IDENTIFY_KNOWLEDGE → GUIDED_SOLVING → COMPLETE |

### 模型选型

| 用户区域 | 模型 | 接入方式 |
|---------|------|---------|
| 国内 | Qwen2.5-VL | DashScope OpenAI 兼容接口 |
| 海外 | Claude Sonnet 4.6 | 已有 Anthropic client |

路由依据：环境变量 `AI_REGION=domestic|international`

- 对话消息 → PostgreSQL（Prisma，已有）
- 图片文件 → 腾讯云 COS
