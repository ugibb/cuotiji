## 第 5 轮

🤖 **AI**

好，这个设计很好——不是强制逐题走，而是学生自主选择深度。三种题型对应三种不同的 AI 互动模式。

梳理一下题目列表的三种状态：

## 最终总结确认

🤖 **AI**

游戏过关——这句话定了整个产品的情感基调。

知识点不是「待修复的漏洞」，而是一关一关要打通的关卡。打通一关有成就感，没打通的关卡看着就想去挑战。

**MVP 页面清单（草稿）**

| 页面 | 核心功能 |
|------|---------|
| 首页 | 训练进度 + 知识关卡地图 + 今日打卡入口 |
| 选章节 | 选择本次练习对应的章节 |
| 拍照上传 | 拍摄纸质试卷 |
| 批改等待 | AI 处理中的过渡动画 |
| 作业汇总 | 得分 + 情绪价值文案 + 进入复盘 |
| 题目列表 | ✓ / ✗ / ? 三种状态列表 |
| 题目详情 | 三种交互模式（对/错/不会）|
| 复盘小结 | 本次成果 + 推送微信通知 |

两个问题：

1. 「知识关卡地图」MVP 阶段要不要做——还是首页先用简单的进度条替代，关卡地图放后续？
2. 导航栏需不需要？如果需要，底部放哪几个 Tab？
