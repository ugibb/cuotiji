# PRD：AI 学习教练（微信小程序 MVP）

> 版本：v2.0
> 日期：2026-04-16
> 阶段：MVP（微信小程序）
> 修订说明：基于竞品分析、失败案例复盘及产品定位讨论，对 v1.0 进行全面重写

---

## 一、产品定位

### 1.1 一句话定义

> 以错题为数据入口，以考试目标为驱动，帮学生系统性弥补知识盲区、达成阶段性学习目标的 **AI 学习教练**。

这不是错题整理工具，是**目标驱动的学习项目管理系统**。

### 1.2 核心竞争力（以终为始）

终态目标：积累「学生 × 知识点 × 时间 × 考试结果」四维数据资产，构建错题模式 → 分数预测模型。

从终态倒推，三项核心竞争力缺一不可：

| 竞争力 | 描述 | 为何难以复制 |
|--------|------|------------|
| **根因诊断** | 从「粗心」标签深挖到真实知识漏洞，通过对话定位错误背后的根本原因 | 需要积累大量「对话 × 知识点 × 结果」训练数据 |
| **知识盲区可视化** | 实时更新的知识点掌握状态，学生和家长随时可看到盲区在收窄 | 依赖 Layer 2 纵向数据积累，不可被外部采购 |
| **目标驱动调度** | 项目制框架（考试日期 + 目标分数）× 遗忘曲线，每日复习有理由、有优先级、有截止感 | 预测模型需要真实考试结果反哺，越用越准 |

### 1.3 目标用户

| 角色 | 描述 | 核心诉求 | 触动点 |
|------|------|---------|--------|
| **学生（主要用户）** | K12 学生，重点初中（兼顾小学高年级） | 知道自己为什么错、今天该复习什么 | 「AI 告诉我真正的原因，不只是说我粗心」 |
| **家长（付费决策者）** | 关注学习效果，焦虑但不想过度介入 | 看到可量化的进步，知道孩子的知识盲区 | 「孩子的盲区一目了然，不靠感觉靠数据」 |

### 1.4 MVP 范围

- **科目**：初中数学（单科垂直切入，验证核心假设后扩科）
- **平台**：一个微信小程序（含学生视图 + 家长视图，角色切换，非两个独立小程序）
- **核心闭环**：拍照录入 → 根因诊断对话 → 知识盲区更新 → 遗忘曲线复习 → 考试项目追踪 → 家长报告

## 二、用户故事

### US-01：极简录入（摩擦归零）

**作为**学生，  
**我想要**拍一张照片就完成错题录入，  
**以便**考完试 10 道错题不超过 5 分钟处理完。

**验收标准：**

- [ ] 相机界面底部提供模式切换：**整页模式**（默认）/ **单题模式**
- [ ] **整页模式**：四角自动检测纸张边缘并吸附，拍摄后自动透视矫正
- [ ] 图片上传至服务器 ≤ 3 秒（AI 处理异步进行，不阻塞用户）
- [ ] AI 完成版面分割后，显示「确认错题」界面：标出检测到的错题区域，用户勾选确认或取消
- [ ] **单题模式**：显示可拖动裁切框，用户手动框住目标题目
- [ ] AI 自动提取 LaTeX 格式题目文本和知识点标签，用户可编辑修正
- [ ] 识别失败时提供便捷手动补充入口，不直接报错拦截流程
- [ ] 整页模式：整份试卷（2–4 页）录入总时长 ≤ 5 分钟

**技术说明：**
- 印刷体题目 OCR 目标准确率 ≥ 95%
- 工整手写 OCR 目标准确率 ≥ 80%
- 草稿/演算过程：不做 OCR，由多模态 LLM 直接做图像理解（用于根因诊断对话的上下文）
- 错题检测（识别老师批改痕迹：红圈/叉/×）：MVP 阶段允许漏检，用户可手动补充

---

### US-02：根因诊断对话（核心差异化）

**作为**学生，  
**我想要**在录入错题后，AI 帮我搞清楚为什么错，而不只是告诉我答案，  
**以便**下次遇到同类题不再犯同样的错误。

**验收标准：**

- [ ] 错题确认后自动触发根因诊断，可选择「现在诊断」或「稍后」
- [ ] AI 通过 3–5 轮对话引导学生描述解题思路（例：「你是怎么列方程的？」）
- [ ] 如学生同时上传了草稿照片，AI 可直接基于草稿图像提问，减少文字输入
- [ ] 诊断结束后，AI 输出根因分类（四选一）：
  - 知识漏洞（概念根本未掌握）
  - 理解偏差（掌握了但理解有偏差）
  - 计算粗心（步骤正确，执行出错）
  - 审题失误（题意理解有误）
- [ ] 根因描述以自然语言保存到错题卡（例：「你把移项时符号变化的规则用反了」）
- [ ] 诊断完成后，AI 给出 1 条针对性学习建议（不超过 2 句话）
- [ ] 小学低年级（可设置）：改用结构化问题模板替代自由对话，降低输入门槛
- [ ] 对话轮次上限 5 轮，超时或无响应自动跳过，标记为「待诊断」

**设计原则：**
- 对话语气友好，不批判，不强调「错了」
- 诊断跳过不惩罚，积累数据比强制完成更重要

---

### US-03：知识盲区地图

**作为**学生，  
**我想要**看到自己各知识点的掌握状态，  
**以便**知道哪些地方是真正的盲区，有目的地补缺。

**验收标准：**

- [ ] 按课程知识点体系展示各节点掌握状态（三态：未掌握 / 复习中 / 已掌握）
- [ ] 至少录入 5 道错题后解锁，避免数据不足时误导
- [ ] 每道错题的根因诊断完成后，对应知识点状态自动更新
- [ ] 点击任意知识点 → 下钻查看该点下所有错题及根因摘要
- [ ] MVP 阶段以**列表 + 色块**展示（红/黄/绿标色），不做复杂星图可视化（v2 迭代）
- [ ] 支持按「当前考试项目」筛选，聚焦当前目标范围

**知识图谱数据来源：**
- Layer 1（静态骨架）：基于教育部初中数学课程标准，参考科大讯飞教育 API 补充细节，一次性建设
- Layer 2（动态状态）：由学生错题数据实时更新，为产品数据护城河所在

---

### US-04：考试项目（目标驱动）

**作为**学生，  
**我想要**设定一次考试的目标分数，并得到 AI 制定的复习计划，  
**以便**有针对性地冲刺，而不是漫无目的地复习。

**验收标准：**

- [ ] 新建考试项目：输入科目 + 考试日期 + 目标分数（提供默认值：历史最高 + 5 分）
- [ ] 创建项目全程 ≤ 2 分钟
- [ ] AI 基于当前知识盲区 + 距考试天数，生成「本周优先修复知识点」（不超过 3 个）
- [ ] 项目进行中：首页顶部显示「距考试 X 天 · 已修复 Y / 待修复 Z 个知识点」
- [ ] 「考后 24 小时」触发推送：「趁记忆还清晰，现在录入错题只需 5 分钟」
- [ ] 考后录入实际分数 → AI 输出简短复盘（哪些知识点预测方向对了，哪些没有）
- [ ] 未解决的盲区知识点自动沿用至下一个项目，螺旋积累

---

### US-05：每日遗忘曲线复习

**作为**学生，  
**我想要**系统告诉我今天该复习哪些题，  
**以便**不依赖自律也能坚持复习。

**验收标准：**

- [ ] 基于 SM-2 间隔重复算法，为每道错题计算下次复习时间
- [ ] 仅在有「到期错题」时推送通知（「今天有 X 道题需要复习，预计 Y 分钟」）；无到期题不推送
- [ ] 首页清晰显示「今日待复习：X 道」；无待复习时显示「本周暂无到期复习，保持得很好」
- [ ] 复习交互（卡片滑动）：
  - 点击卡片正面：查看题目
  - 点击翻转：显示正确答案 + 根因摘要
  - **右滑（扔掉）**：「今天掌握了」→ 物理惯性动效飞出 + 轻微音效
  - **左滑（留下）**：「还没会」→ 卡片回到牌堆底部，缩短下次复习间隔
  - **上滑（标记疑问）**：「有问题」→ 进入待解决列表
- [ ] 右滑后提示：「已掌握！X 天后再巩固一次」（避免用户误以为永久删除）
- [ ] 日常作业/练习错题同样通过 US-01 流程录入，补充复习队列，维持日活

**冷启动处理：**
- 注册后引导「快速导入近期错题」（上次考试 / 上周作业），快速填充初始复习队列
- 前两周复习队列较少时，优先推送当前考试项目中优先级最高的盲区题

---

### US-06：家长报告（续费杠杆）

**作为**家长，  
**我想要**每周收到孩子的学习状况报告，  
**以便**了解知识盲区的变化趋势，判断产品是否有效。

**验收标准：**

- [ ] 每周一早 8 点通过**微信服务消息**推送（家长无需主动打开小程序）
- [ ] 报告回答三个核心问题：
  - **现在在哪里**：本周新增 X 道错题，已掌握 Y 道，TOP3 盲区知识点
  - **进步了多少**：与上周对比，盲区知识点减少 N 个
  - **下一步做什么**：本周建议重点关注的 1–2 个知识点
- [ ] 家长点击报告 → 进入小程序家长视图（只读），可查看盲区列表和统计趋势
- [ ] 家长视图不展示具体题目内容（保护学生隐私，避免监控感）
- [ ] 账号体系：家长创建家庭账户 → 添加孩子档案 → 生成关联码 → 孩子微信扫码绑定
- [ ] 一个家长账户支持关联多个孩子

---

### US-07：AI 识别纠错（数据质量保障）

**作为**学生，  
**我想要**在 AI 识别结果有误时能快速修正，  
**以便**保证错题记录准确，后续分析不偏差。

**验收标准：**

- [ ] 识别结果展示后，可点击任意字段进行编辑（题目文本、知识点标签、根因）
- [ ] 知识点标签支持手动添加/删除，从知识图谱节点中选择
- [ ] 识别完全失败时，跳转简化手动录入界面（仅需选知识点 + 简短描述），不强制 OCR
- [ ] 用户每次纠错行为记录为训练反馈数据，持续改进模型

## 四、数据结构（系统侧 Schema）

错题记录为核心数据实体，字段分三层：

```
WrongQuestion（错题记录）

── 原始输入（用户提供）
   ├── question_image_url       原题照片
   ├── work_image_url           草稿照片（可选）
   ├── source_type              来源（期中考 / 期末考 / 日常作业 / 练习）
   ├── subject                  科目
   └── created_at               录入时间

── AI 派生字段（自动填充，用户可纠错）
   ├── question_latex           LaTeX 格式题目文本
   ├── correct_answer           正确答案
   ├── student_answer           学生答案（如可识别）
   ├── knowledge_point_ids[]    知识点标签（映射至知识图谱节点 ID）
   ├── error_type               错误类型（knowledge_gap / misunderstanding / careless / misread）
   └── root_cause_summary       根因描述（自然语言）

── 复习管理（算法维护）
   ├── mastery_level            掌握程度（0–5，SM-2 算法维护）
   ├── review_count             已复习次数
   ├── last_reviewed_at         上次复习时间
   ├── next_review_at           下次计划复习时间
   └── review_history[]         历次复习结果（mastered / not_mastered / flagged）
```

## 五、非功能性需求

### 5.1 性能
- 图片上传完成 ≤ 3 秒（4G 网络）
- AI 版面分割结果返回 ≤ 10 秒（异步，不阻塞录入流程）
- 首页加载 ≤ 2 秒
- 根因诊断每轮对话响应 ≤ 5 秒

### 5.2 识别准确率目标
- 印刷体 OCR ≥ 95%
- 工整手写 OCR ≥ 80%
- 草稿理解：多模态 LLM 语义理解，作为对话上下文参考，不要求精确文字转录

### 5.3 数据安全与合规
- 学生数据不对外共享，不用于广告
- 未成年人数据需家长授权（账号绑定时获取）
- 图片识别完成后压缩存储，原图保留 90 天后清除
- 遵守《未成年人网络保护条例》
- 不展示任何广告内容（MVP 阶段）
- 小程序每日使用提示（超 40 分钟提示休息）

## 六、商业模式

### 成本基准（单次完整 AI 诊断路径）

| 环节 | 估算成本 |
|------|---------|
| OCR + LaTeX 标准化 | ¥0.03–0.08 |
| 多模态 LLM 草稿理解 | ¥0.10–0.30 |
| 根因诊断对话（3–5 轮） | ¥0.10–0.20 |
| 练习建议生成 | ¥0.05–0.10 |
| **合计** | **¥0.30–0.70（保守取 ¥0.50）** |

降本杠杆：热门题目缓存（同一试卷被多人上传时，AI 处理成本仅计一次）

> 上线前需用 10–20 道真实错题完整跑一遍流程，以实测数据替换上述估算值。

### 定价策略

| 版本 | 价格 | 功能边界 |
|------|------|---------|
| **免费版** | 免费 | OCR 录入 + 基础存储（每月上限 20 道）；不含根因诊断、盲区地图、考试项目 |
| **Pro 版** | ¥49 / 月 或 ¥399 / 年 | 完整根因诊断 · 无限存储 · 盲区地图 · 考试项目 · 家长周报 |

> 付费逻辑：不是「功能值不值这个钱」，而是「考好了这点钱算什么」

### MVP 阶段（0–3 个月）：完全免费，验证三个核心假设

| 假设 | 验证指标 | 目标值 |
|------|---------|--------|
| 根因诊断真的有用 | 根因诊断对话完成率 | ≥ 60% |
| 「拍照」能成为习惯 | 考后 24h 录入率 | ≥ 40% |
| 家长感知到价值 | 家长周报打开率 | ≥ 50% |

辅助观察指标：
- 复习执行率（到期复习完成比例）≥ 35%
- 考试周期完成率（完整走完一个项目）≥ 25%
- D30 留存率 ≥ 20%

## 七、设计原则

1. **用户只做一件事**：拍照。其余全由 AI 完成，摩擦归零
2. **驱动力来自真实进步**：满足感来源于「AI 告诉我为什么错了」「这道题今天真的会了」，而非打卡数字
3. **嵌入已有仪式**：「考后 24 小时录入」「有到期题才推送复习」，不创造新行为，借力已有节点
4. **家长感知价值，学生感知成长**：两者触动点不同，分开设计
5. **允许不完美**：识别允许有误，纠错比拦截更重要；诊断允许跳过，积累数据比强制完成更重要
