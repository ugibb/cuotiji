## 一、背景与诉求

当前 `chat-input` 组件的语音功能只录音并上传音频文件，UI 过于简陋，缺少录音状态反馈和文字预览区。

**核心体验目标**：

- 参考微信/豆包语音输入：点击麦克风 → 弹出语音面板 → 说话 → 文字实时出现 → 自动或手动停止后文字填入输入框
- 高准确率，支持方言（尤其粤语、川普等）
- 响应快，停止后文字延迟 ≤ 1s

## 二、微信小程序官方能力现状

| 官方 API | 能做 | 不能做 |
| --- | --- | --- |
| `RecorderManager` | 录音（MP3 / PCM / AAC）、音量监控 | **不含 STT** |
| `voice_recognition` 录音模式 | 优化麦克风采集质量（Android only） | 无转写能力 |
| 微信聊天语音条 | 腾讯内部 ASR（对话功能） | **不对外暴露 API** |

**结论：微信小程序没有官方语音转文字组件，开发者必须接入第三方 STT 服务。**

## 四、开源 / 免费 STT 方案

### 4.1 方案总览

| 方案 | 模型 | 中文准确率 | 方言 | 部署方式 | GPU 依赖 | 延迟（CPU） | 费用 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| **FunASR + Paraformer-zh** | 阿里 DAMO | ★★★★★ | 粤语（单独模型） | 自部署 Docker | 可纯 CPU | 300~800ms | **完全免费** |
| **faster-whisper** | OpenAI Whisper 优化版 | ★★★★ | 依模型大小 | 自部署 | 可纯 CPU（慢） | 1~3s（medium） | **完全免费** |
| **SenseVoice-Small** | 阿里开源 | ★★★★ | 粤语、闽南等 | 自部署 | 可纯 CPU | < 500ms | **完全免费** |
| **Groq Whisper API** ✅ | Whisper large-v3-turbo | ★★★★ | 多语言 | 云 API | N/A（托管） | < 300ms | **免费额度 100h/月** |
| **OpenAI Whisper API** | Whisper large-v3 | ★★★★ | 多语言 | 云 API | N/A（托管） | < 500ms | $0.006/min（约¥0.06） |

### 4.2 重点方案详解

#### ① FunASR + Paraformer-zh（推荐自部署方案）

> 阿里 DAMO 开源，GitHub 12k stars，中文学术界公认最强开源 ASR

- **模型**：`paraformer-zh`（普通话）+ `paraformer-zh-streaming`（流式版）
- **方言**：需搭配 `paraformer-zh-dialect` 或 `cantonese` 模型（粤语单独一个模型）
- **部署**：官方提供 Docker 镜像，支持 REST API 和 WebSocket

```bash
# 一键启动（REST 模式，CPU 可用）
docker run -p 10095:10095 \
  registry.cn-hangzhou.aliyuncs.com/modelscope-repo/funasr:latest \
  --model paraformer-zh \
  --model-revision v2.0.4
```

- **接入方式**：服务端调 `POST http://localhost:10095/api/v1/asr`，返回 JSON 文本
- **硬件需求**：4 核 CPU / 4GB 内存可运行（现有服务器 123.207.64.65 可直接部署）
- **缺点**：方言需要单独切换模型；首次加载模型约 30s

#### ② Groq Whisper API ✅ 当前选型

> Groq 用定制 LPU 芯片运行 Whisper，速度是 OpenAI 原版的 10~50x

- **免费额度**：100 分钟/月（足够早期用户测试，超出 $0.04/1000s）
- **接口**：兼容 OpenAI 格式，直接替换 baseURL

```typescript
// 服务端：上传 MP3，调 Groq 转写
const transcript = await groq.audio.transcriptions.create({
  file: fs.createReadStream(mp3Path),
  model: 'whisper-large-v3-turbo',
  language: 'zh',
})
// transcript.text → 转写结果
```

- **优点**：无需自部署，速度极快（< 300ms），无需服务器资源
- **缺点**：方言支持不如讯飞；免费额度有上限；需访问境外 API（需测试延迟）
- **切换路径**：后端改一个函数即可升级到 FunASR 自部署或讯飞，小程序端零改动

#### ③ SenseVoice-Small（轻量自部署备选）

> 阿里开源，模型体积仅 234MB，CPU 推理 < 500ms，支持情绪识别

- **特点**：多语言（中/英/日/粤/闽南），比 Whisper small 准确率高
- **部署**：可通过 `funasr` 框架加载，或直接用 Python `modelscope` 推理
- **适合**：服务器资源有限但想自部署的场景

### 4.3 开源方案对比小结

```text
高准确率 + 方言覆盖  →  FunASR Paraformer-zh（自部署，免费）
零部署成本 + 够快    →  Groq Whisper API（免费额度内）✅ 当前选型
轻量 + 自部署        →  SenseVoice-Small
```

## 六、架构方案（已确认）

### 选型：录完上传 + 服务端中转

```text
用户点击麦克风图标
    │
    ▼
语音面板弹出（参考微信/豆包风格）
RecorderManager.start({ format:'mp3', sampleRate:16000 })
    │  waveform 随 onFrameRecorded.volume 实时动效
    │
用户点击停止 / 静音超过 2s 自动停止
    │
    ▼
RecorderManager.stop()  ← onStop 拿到 tempFilePath
面板显示"识别中..."
    │
    ▼
wx.uploadFile → 服务端 POST /api/stt   ← 密钥在服务端，不暴露
    │
    ▼
Groq Whisper large-v3-turbo 转写
返回 { text: "我认为应该先求公因数..." }
    │
    ▼
文字填入底部输入框（panel 收起）
用户可编辑后点击发送
```

> 服务端 `/api/stt` 内部调 Groq API，切换 STT 服务商时只改此文件，小程序端零改动。

## 七、UI 重设计（参考微信 / 豆包）

### 微信语音键盘 / 豆包语音输入 核心交互模式

| 交互要素 | 微信语音键盘 | 豆包 | 本项目实现 |
| --- | --- | --- | --- |
| 触发方式 | 点击键盘内麦克风图标 | 点击输入框旁麦克风 | 点击输入框右侧麦克风图标 |
| 录音方式 | 自动检测静音停止 | 点击停止按钮 | 点击停止 + 超时自动停止 |
| 录音 UI | 底部弹出半屏面板 | 底部弹出半屏面板 | 底部弹出半屏面板 |
| 波形展示 | 中央动态声波 | 3~5 条竖向脉冲柱 | 15 条竖向脉冲柱 |
| 文字预览 | 实时字幕（流式） | 实时字幕（流式） | 停止后填入输入框（非流式） |
| 完成动作 | 文字填入输入框 | 文字填入输入框 | 文字填入输入框 |
| 取消方式 | 滑动关闭 / 点取消 | 点取消 | 点取消按钮 |

> 本项目使用 Groq 文件上传（非流式），无法做实时字幕；停止后 < 1s 出文字，体验可接受。后续升级讯飞流式 SDK 时可补上实时字幕。

### 关键视觉细节

- **面板**：`border-radius: 24rpx 24rpx 0 0`，白色背景，顶部 drag handle（40×4rpx 灰条）
- **进入动效**：`transform: translateY(100%)` → `translateY(0)`，300ms ease-out
- **波形**：15 条柱，宽 6rpx，间距 8rpx，圆角，颜色 `#4845a4`；高度范围 8~72rpx；音量映射为高度，有 `transition: height 80ms ease`
- **停止按钮**：深色圆角矩形，点击后立即变为灰色 disabled 状态
- **取消**：灰色文字按钮，点击后面板立即收起，不发送
- **识别中**：波形柱全部降至最低，显示三点 loading 动效
- **成功**：面板以 200ms 滑出动效收起，文字出现在输入框

## 八、服务端改造（最小化）

新增一个纯转写路由，对小程序透明：

```text
POST /api/stt
Body: multipart/form-data { voice: <mp3 file> }
Response: { success: true, data: { text: string } }
```

内部对接 Groq Whisper API，切换 STT 服务商时只改此文件。

后续 `voice-dialogue` 路由可改为接收 `text` 而非音频文件，逻辑更清晰。

## 十一、已确认决策

| 事项 | 决策 |
| --- | --- |
| STT 服务商 | **Groq Whisper**（开发阶段），视需求升级 FunASR / 讯飞 |
| UI 交互风格 | **参考微信 / 豆包**：点击触发、底部半屏面板、波形动效、停止后填入输入框 |

| 实时字幕 | 当前不做（Groq 为文件上传非流式）；升级讯飞流式 SDK 后可补 |
