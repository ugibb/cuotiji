### 技术方案总结

**核心挑战确认**

扫描件 + 数学公式 + 中文 + 结构化分段，这是难度最高的 PDF 处理场景。方案分两层：**OCR/LaTeX 提取** + **结构解析入库**。

---

**推荐方案：Mathpix + Claude（付费，效果最佳）**

```
PDF
  ↓ PyMuPDF（拆页 + 提取图片）
  ↓ Mathpix API（扫描图 → Markdown + LaTeX）
  ↓ Claude API（结构解析 → JSON）
  ↓ 写入 pre_ 表
```

**第一步：Mathpix 处理每页**
- 输入：每页扫描图
- 输出：Markdown，数学公式自动转 LaTeX，中文支持良好
- 费用：约 $10/1000 页，一套教程估计 300-600 页，成本 $3-6
- 优势：目前公认数学 OCR 精度最高的服务

**第二步：Claude API 结构解析**
- 输入：Mathpix 输出的 Markdown
- Prompt 指令：按「知识导引/知识点/好方法/举一反三/探索题」分段切割，输出 JSON
- 字段映射直接对应 `pre_lecture_kp`、`pre_lecture_methods`、`pre_questions`
- 费用极低（Haiku 即可，每页不到 $0.001）

**图片处理（分类存储）：**
- PyMuPDF 提取嵌入图片
- 按位置判断类型：题目附图 / 思维导图 / 二维码（二维码可用 `pyzbar` 解码留存原始 URL）
- 上传 COS/OSS，URL 存 `pre_questions.stem_raw` 或专用字段

---

**备选方案：Claude Vision 一体化（付费，更简单）**

```
PDF → 页面转图片 → Claude Vision（OCR + 结构解析一次完成）→ JSON → pre_ 表
```

- **优点**：一个 API 搞定所有事，Prompt 可直接输出符合 pre_ 表结构的 JSON
- **缺点**：复杂数学公式的 LaTeX 精度略低于 Mathpix
- **成本**：Sonnet ~$0.02/页，500 页约 $10；Haiku 约 $3-5

---

**免费方案：GOT-OCR 2.0 + Claude（需本地 GPU）**

- GOT-OCR2.0：2024 年新模型，支持中文 + 数学 LaTeX，效果接近 Mathpix
- 需要 GPU（最低 RTX 3090 16GB），本地跑
- OCR 出来的文本再接 Claude 做结构解析

---

**推荐优先顺序**

| 场景 | 方案 |
|------|------|
| 先跑通流程，追求最高质量 | **Mathpix + Claude** |
| 想简单快速，质量够用 | **Claude Vision 一体化** |
| 长期批量处理，有 GPU | **GOT-OCR2.0 + Claude** |

建议先用 Claude Vision 一体化跑 2-3 个 PDF 验证效果，如果数学公式精度不够再切 Mathpix。

## 第 10 轮

🤖 **AI**

你的直觉是对的——**Mathpix 不需要你预先切题**。

它内部会先做 Layout Analysis（区域检测），自动识别文字块、公式块、图片块、表格，然后分区域识别，最后拼成 Markdown。你送进去一整页扫描图，它自己会处理分割逻辑。

**Claude Vision 也类似**——只要你 Prompt 写清楚「每道题单独输出一个 JSON 对象」，它能在一次调用里完成切割 + 识别 + 结构化，不需要外部裁图。

**切题真正有价值的场景是：**

| 场景 | 是否值得切 |
|------|----------|
| 页面密度极高（一页 5 道以上） | 值得，避免 LLM 上下文混淆 |
| 需要精确知道「这张图属于第几题」 | 值得，位置对应关系更清晰 |
| 用的是弱 OCR 工具（Tesseract 等） | 必须切，否则识别乱 |
| Mathpix / Claude Vision | 不需要，内部已处理 |

对于学而思秘籍这种每页 1-2 道题的排版密度，**跳过切题直接送整页给 Mathpix 或 Claude Vision 就够了**，不要增加不必要的复杂度。

**Claude 生成 TikZ 代码 → Python `pdflatex` 编译成 PNG → 上传 CDN → URL 存入 `knl_question_solutions.solution_latex` 或新增 `diagram_url` 字段**
