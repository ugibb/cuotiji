## 020-competitor-scan.md

**[备注 1]** 「错题本厚厚一本，翻出来也不知道从哪复习」

- 用户原始痛点：信息过载 + 无从下手
- 产品方向：AI 自动定位 & 归类，让知识点没有盲区
- 讨论结论：这是一个核心卖点，但需拆成两层表达：
  - 对学生：「不用整理，AI 帮你找到今天最该复习的那道题」
  - 对家长：「孩子的知识盲区一目了然，不靠感觉靠数据」
- 优化方向：PRD 中卖点描述需要区分学生视角 vs 家长视角，两者的触动点不同

---

**[备注 2]** 「录入太麻烦，考完试拍十几道题要半小时」

- 用户原始痛点：输入摩擦成本太高，劝退用户
- 产品方向：用户只做「拍照」一步，其余全由 AI 完成
- 完整流程设计：
  1. 拍照上传（用户唯一操作）
  2. OCR 识别（矫正/补光/定位）→ 题目 LaTeX 标准化
  3. 学生解题思路意图识别（需同时拍草稿/作答过程）
  4. 对话方式协助学生定位错误背后真实原因
  5. 完成错题登记
  6. 生成针对性练习题
  7. 登记下一次复习计划
- 技术可行性评估：大部分步骤有现成 API，核心壁垒在步骤 3
- 优化方向：步骤 3「意图识别」是最大技术壁垒，也是最大差异化优势，需重点规划
- 补充讨论：小学生手写 & 草稿乱涂乱画的识别可行性
  - 印刷题目 OCR：准确率 95%+，可靠
  - 学生最终答案 OCR：85-90%，可用
  - 演算草稿：不应用 OCR，改用多模态 LLM 直接做图像理解（GPT-4o / Claude Vision）
  - 小学低年级特殊处理：题型集中，改用对话诊断 + 结构化错误模板替代草稿识别
  - 结论：草稿识别的技术门槛恰好形成竞争壁垒，纯 OCR 方案的竞品难以复制

**[备注 3]** 「每日10分钟打卡 + 仿Duolingo连击机制」

- 用户反馈：交互感觉不友好，不希望人为增加使用频率
- 核心问题：Duolingo 机制是外部驱动（连击焦虑/社交压力），错题复习需要的是内部驱动（真实进步感）
- 决策：放弃定时触达，改为「定向触达」
  - 触发时机：遗忘曲线算法预测「快忘了」时推送，而非每天固定时间
  - 完成感来源：「这道题已掌握」而非「连击+1」
  - 没有待复习时：直接告知「本周没有到期复习」，不强行填充内容
- 家长可见内容调整：从「使用行为数据（打卡天数）」改为「学习掌握数据（盲区收窄、掌握率变化）」
- 优化方向：PRD 中删除连击/打卡机制，改为基于遗忘曲线的精准推送设计

**[备注 4]** 「本周错题分析报告 → 知识星图可视化」

- 用户想法：基于知识体系的「上帝视角」，可视化展示学生各知识点掌握状态，有目的性补短板
- 核心问题：知识体系是否需要重运营从头积累？还是可以借助外部资源？
- 结论：知识体系分两层，策略不同
  - Layer 1（知识图谱骨架/静态）：可借外力，不重运营
    - 教育部课程标准（公共资源）做骨架
    - 科大讯飞教育 API 补细节（最开放的巨头）
    - 一次性建设 + 随课标更新小幅维护
  - Layer 2（学生掌握度状态/动态）：必须自己积累，这是真正的护城河
    - 错误共现模式（在知识点 A 错的学生，73% 同时在 B 有盲区）
    - 知识点依赖修复路径（掌握 C 之前必须修复 D）
    - 任何外部资源都无法替代，越用越强的数据飞轮
- 与巨头合作的正确姿势：买 Layer 1 的 API，用学校/教培机构合作换冷启动数据积累 Layer 2
- 优化方向：PRD 中星图功能描述需强调 Layer 2 数据的积累路径，这是产品长期壁垒

**[备注 5]** 「免费版：每月最多保存 30 道错题」定价需基于成本核算

- 用户关注点：需先核算错题登记单次成本，再推算定价和运营目标
- 成本估算框架（完整 AI 诊断路径）：
  - OCR + LaTeX：¥0.03–0.08 / 题
  - 草稿图像理解（多模态 LLM）：¥0.1–0.3 / 题
  - 对话式根因诊断（3–5 轮）：¥0.1–0.2 / 题
  - 练习题生成：¥0.05–0.1 / 题
  - 合计：**¥0.3–0.7 / 题（保守取 ¥0.5）**
- 关键结论：
  - 免费版必须限制在轻量路径（仅 OCR + 存储，~¥0.1/题），否则不可持续
  - 完整 AI 诊断作为付费特权，这也是付费墙的产品设计依据
  - 初步定价参考：¥49/月（覆盖 ~40 道完整路径，60% 毛利率）
  - 降本杠杆：热门题目缓存（同一试卷被多人上传时，AI 成本分摊）
- 优化方向：PRD 开发前需做小规模 API 成本实测（10–20 道真实错题跑完整流程）

**[备注 6]** 「习惯养成深度 + 家长报告体系」作为核心护城河

- 用户认同方向，追问：具体怎么做到？
- 习惯养成深度的实现路径：
  - 核心逻辑：不创造新行为，嵌入已有学习仪式（考后、作业发下来、考前）
  - 「考后 24 小时」机制：考试日后主动推送，趁记忆清晰时录入错题
  - 学校日历集成：考试节点自动调整复习强度
  - 进度锚点：「本学期录入 47 题，已掌握 38 题」→ 学习档案认同感
  - 目标：学生拿到考卷的第一个动作是打开产品拍照
- 家长报告体系的实现路径：
  - 不做使用数据（打卡天数），做学习掌握数据（盲区变化、进步幅度、下一步建议）
  - 三个核心问题：现在在哪里？进步了多少？下一步做什么？
  - 护城河来源：报告格式可复制，背后 6 个月的掌握度数据不可迁移
- 两者的复利结构：习惯养成 → 数据积累 → 报告质量提升 → 家长续费 → 更多用户 → 数据飞轮
- MVP 验证策略：先手工做第一批家长报告，验证「收到实质性报告的用户续费率是否显著更高」

## 030-failure-analysis.md

**[备注 7]** 「错题收录格式尚需优化」——格式是什么？业内有标准吗？

- 问题拆分：格式分两层
  - 用户侧：录入交互（极简，只拍照）
  - 系统侧：数据 schema（由 AI 自动派生，字段丰富）
- 业内标准：无统一标准，数学公式用 LaTeX（有共识），其余自行设计
- 建议的系统侧数据结构：
  - 原始输入：题目照片、草稿照片、来源、录入时间
  - AI 派生字段：LaTeX 题目、正确答案、学生答案、知识点标签[]、错误类型、根因描述
  - 复习管理：掌握程度、复习次数、上次/下次复习时间、历次复习记录
- 核心难点：知识点标签体系（knowledge_points[]）的粒度和分类标准
  - 直接决定星图质量、相似题推送、家长报告的有效性
  - 必须与 Layer 1 知识图谱骨架同步设计，不能单独定义
- 优化方向：PRD 技术设计章节需专门描述数据 schema，知识点标签体系与知识图谱共同规划

**[备注 8]** 「课外资料无用感」→ 引出「项目制学习」框架

- 用户洞察：学习机内容无用感 = 内容选择没有目标锚点，推送逻辑是「你学了什么」而非「你差什么」
- 项目制学习框架：
  - 项目启动：考试日期 + 目标分数 → AI 诊断当前水平 → 生成冲刺计划
  - 项目执行：错题录入持续更新 → 按计划推送练习 → 进度可视化（差距收窄）
  - 项目收尾：实际分数输入 → 复盘预测 vs 实际 → 未解决差距带入下一项目
  - 螺旋上升：每次考试结果反哺预测模型，越来越准
- 产品重新定位：从「智能错题本（工具）」→「AI 考试准备教练（目标驱动）」
  - 付费逻辑从「功能值不值钱」变为「考好了这点钱算什么」
  - 家长感知从「整理错题」变为「孩子有计划地准备考试」
- 护城河升级：每个学生完成的项目越多，「错题分布 → 实际分数」预测越准，单用户纵向价值越高
- 设计挑战：项目启动摩擦成本，需控制在 2 分钟以内（默认目标分数 + 用历史错题代替初始诊断）
- 优化方向：PRD 需重新审视产品定位，以「项目制学习」作为核心产品框架重新组织功能

**[备注 9]** 「独立错题 App 没有高频场景锚定，是硬伤」

- 用户关注：这是产品设计和运营需要重点关注的核心指标
- 问题本质：使用触发点稀少（月 2–4 次），用户感知不到价值 → 不续费
- 三条应对路径：
  1. 不争频率，争「时刻密度」：把「考后 24 小时」做成极强仪式感，一次高价值胜过多次低价值
  2. 「环境存在感」替代「主动打开频率」：桌面 Widget、锁屏通知、家长微信推送
  3. 家长频率驱动学生频率：家长高频查看报告 → 提醒孩子 → 社会压力触发使用
  4. 项目制潮汐式高频：考前 2 周 + 考后 48 小时天然高频，无需人为制造
- 最重要的认知转变：换掉错误的 KPI
  - 错误：DAU/MAU、日均时长、次日留存
  - 正确：考试周期完成率、考后 48h 录入率、项目复盘完成率、按学期续订率
- 优化方向：运营目标体系需从「日活指标」重构为「考试周期价值指标」，产品文档中的成功标准需同步更新
- 补充修正（备注 10）：日活不能放弃，低频 + 纯靠自律是已被验证的失败路径
  - 解法：把错题来源从「考试」扩展到「每日作业/练习」，触发频率从月 2–4 次 → 每日
  - 日活两个轻量动作：① 30 秒拍照录入作业错题 ② 遗忘曲线驱动 2 分钟每日复习推送
  - 冷启动问题：前两周积累量少，推送稀疏，需引导用户批量录入历史错题快速填充初始库
  - 最终指标体系：日活（复习执行率）+ 考试周期完成率，两者并重，分别衡量不同维度

## 010-prd.md

**[备注 11]** 「微信小程序学生端 + 家长端」——为何要区分两个版本？

- 用户质疑：小学生通常由家长注册后交给孩子使用，分两个版本制造摩擦
- 按年龄段看：小学低年级共用家长设备；初中以上才有独立设备
- 建议架构调整：一个小程序 + 角色视图切换（家长视图 / 学生视图）
  - 家长创建家庭账户 → 添加孩子档案 → 视图按角色切换
  - 共用设备场景天然支持，开发成本减半
- 家长报告的更轻方式：每周定时推送微信消息，家长无需主动打开小程序
- 初中以上的分离场景：学生独立登录 + 家长关联码绑定，权限分层（家长只看汇总，不看具体题目）
- 优化方向：PRD 产品形态章节需修改，从「两个小程序」改为「一个小程序 + 角色权限体系」

---

**[备注 12]** 「拍照录入」的最小单位：一道题 vs 一张试卷

- 关键决策：录入单位直接决定摩擦成本
- 结论：以「整页/整张试卷」为录入单位，而非单题
  - 用户操作：拍整页（2–4 张完成整份试卷），≤ 3 秒上传
  - AI 后台：版面分割 → 错题检测（识别批改痕迹）→ 逐题提取（异步，不阻塞用户）
  - 用户确认：AI 标记检测到的错题 → 用户一键确认或修正
- 技术关键：「错题识别」需识别老师批改痕迹（红圈/叉/×），多模态 LLM 可做，需实测精度
- 不同来源处理方式：
  - 试卷（A3/A4）：整页拍 → AI 提取
  - 日常作业纸：同上
  - 练习册（密集排版）：整页拍，需更精细分割
  - 手写错题本：按格分割，最简单
- 优化方向：PRD 中「拍照录入」功能描述需明确以「整页」为单位，AI 异步处理，用户体验无等待感
- 补充（备注 13）：相机 UI 增加模式切换交互
  - 整页模式（默认）：四角自动吸附纸张边缘 + 透视矫正，适合考后处理整张试卷
  - 单题模式：可拖动裁切框，用户手动框住单题，适合日常作业零散录入
  - 拍完整页后增加「确认选择」界面：AI 标记检测到的错题 → 用户勾选确认（同时训练模型）
  - v2 进阶：取景框实时显示题目边界检测（v1 暂不做，避免延迟）

---

**[备注 14]** 「卡片翻转复习」交互设计

- 用户洞察：交互很重要，斯卡片、扔掉的手势体验
- 设计逻辑：物理动作强化记忆 + 牌堆缩减可见 + 扔掉的仪式感
- 交互方案：
  - 点击翻转：正面题目 → 背面答案+解析
  - 右滑扔掉：「掌握了」→ 物理惯性动画飞出 + 轻微音效
  - 左滑留下：「还没会」→ 回到牌堆底部
  - 上滑标记：「有疑问」→ 稍后再看
- 重要认知校准：「扔掉 = 今天毕业，不是永久删除」
  - 扔掉后需提示：「7 天后会再次提醒巩固」
  - 否则用户看到已扔题目重新出现会误以为 bug
- 价值观一致性：满足感来自「真的会了」，而非积分徽章，符合内部驱动力设计原则
- 优化方向：PRD 复习功能章节需专门描述卡片手势交互和动效规范

## 待优化方向汇总

### PRD 需要修改的核心内容

1. **产品定位**：从「错题整理习惯工具」→「AI 学习教练（目标驱动）」
2. **移除**：打卡/连击机制（US-04 整体删除）
3. **移除**：两个小程序，改为一个小程序 + 角色切换
4. **新增**：根因诊断对话模块（核心差异化，MVP 必须有）
5. **新增**：考试项目模块（项目制框架，驱动力来源）
6. **调整**：定价从 ¥9.9/月 → ¥49/月 区间（成本核算支撑）
7. **调整**：KPI 从打卡完成率 → 根因诊断完成率 + 考后录入率 + 家长报告打开率
8. **调整**：相机 UI 增加整页/单题模式切换
9. **调整**：知识盲区展示先用列表（MVP），星图可视化 v2
10. **调整**：家长报告改为微信模板消息推送，无需进入家长端小程序

### 三大核心竞争力（以终为始）

1. **根因诊断能力**：从「粗心」标签深挖到真实知识漏洞
2. **知识盲区可视化**：实时更新的知识点掌握状态，数据护城河
3. **目标驱动复习调度**：项目制 × 遗忘曲线，每日复习有理由有截止感

### MVP 核心验证假设

1. 根因诊断对话完成率（用户愿不愿意配合 AI 问答）
2. 考后 24h 录入率（「拍照」有没有成为习惯）
3. 家长报告打开率（家长看了有没有续费/推荐）

## 产品定义总结（通读完成后）

**一句话定义：**
以错题为数据入口，以考试目标为驱动，帮学生系统性弥补知识盲区、达成阶段性学习目标的 AI 学习教练。

**核心用户：** K12 学生（重点初中生）+ 付费决策者家长

**六层解决方案：**
- 录入层：整页拍照 → AI 自动分割，3 秒完成
- 诊断层：对话式根因诊断，打破「粗心」标签
- 规划层：项目制学习（目标分数 → 冲刺计划）
- 执行层：遗忘曲线自动调度 + 日常作业捕捉
- 感知层：知识星图，盲区收窄可见
- 关系层：家长每周报告，掌握度数据化

**商业机会三阶段：**
1. 近期：订阅制（¥49/月，按学期续订率为核心指标）
2. 中期：数据护城河 → 学情分析 SaaS（B 端）
3. 长期：学习档案 → 个性化辅导精准匹配平台

**一句话商业判断：**
错题本是切入口，知识星图是护城河，项目制是留存飞轮，家长报告是付费杠杆。最终价值在「学生 × 知识点 × 时间」三维数据资产。
