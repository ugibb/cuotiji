## 一、全局视角

### 核心命题

这 6 个月只做一件事：

> **用 50 万，换到「500 注册 / 50 付费 / CAC < 200 元 / 自发转介绍率 > 30%」这组数字。**

这组数字不是财务目标，是打开下一轮融资的钥匙。

### 技术架构总览

```
技术栈
├── 前端：微信小程序（TypeScript/WXML）
├── 后端：NestJS（TypeScript）+ Prisma ORM
├── 数据库：PostgreSQL
├── AI：Claude API（Sonnet 处理复杂推理，Haiku 处理高频轻量任务）
└── 推送：微信订阅消息

代码路径
├── 04-code/server/        NestJS 后端
└── 04-code/miniprogram/   微信小程序
```

### 项目状态机

```
intake → assessing → planning → confirmed → executing → retrospecting → completed
  ↑问卷完成  ↑评估完成   ↑计划生成   ↑计划确认    ↑里程碑完成    ↑复盘提交
```

### 执行框架

```
定位：锁定用户、锁定场景、锁定 MVP 范围
  ↓
目标：确定 6 个月验证目标 + 每个里程碑的成功标准
  ↓
计划：分 5 条并行线推进（产品 / 内容 / 用户 / 增长 / 融资）
  ↓
执行：按里程碑节奏推进，每 2 周一个检查点
  ↓
复盘：每个里程碑结束强制复盘，调整下一阶段节奏
```

### M1 · 第一信任节点上线
**时间：Week 3–5（2026-05-15 ~ 2026-06-04）**

#### 核心目标
> 诊断套题上线，能力报告生成，5 个种子用户完整体验「定位 → 目标 → 评估」路径，获得家长第一轮真实反馈。

#### 背景
能力报告是整个产品的「零时刻」（Zero Moment of Truth）。家长看到这份报告，才真正相信产品。**报告须包含具体数字 + 对比参照 + 行动指引，三要素缺一不可。**

> 示例：「你在余数类题目正确率为 62%，低于同级均值 23%，这是重点突破区」

---

#### 技术任务

**后端：Prisma Schema 扩展（M1 新增表）**
```prisma

```
- [ ] 运行 `prisma migrate` 生成 M1 迁移文件
- [ ] 诊断题数据录入：低/中/高 3 套，每套 15-20 题，每题关联 `knowledgeNodeId`

**后端：M1 API 实现**
```
POST /projects/:id/assessment   提交答题 → AI 批改 → 生成报告 → status: planning
POST /projects/:id/calibrate    能力校正对话 → 更新 weakPoints
```
- [ ] `assessment.controller.ts` + `assessment.service.ts`
- [ ] AI 批改逻辑：每题调用 Haiku 判对错 + 知识点定位
- [ ] 报告生成逻辑：汇总判题结果 → 调用 Sonnet 生成结构化报告 JSON

**后端：AI Prompts（M1）**
- [ ] `ai/grading.prompt.ts`：单题判题（Haiku），输出 `{correct, knowledgeNodeId, explanation}`
- [ ] `ai/ability-report.prompt.ts`：能力报告生成（Sonnet + Prompt Cache），输出：
  ```json
  {
    "abilityLevel": "intermediate",

    "reportSummary": "...",
    "actionableInsights": ["..."]
  }
  ```
- [ ] `ai/calibration.prompt.ts`：校正对话（Sonnet，Socratic 风格），输出 `{updatedWeakPoints, response, isComplete}`

**前端：M1 页面**
- [ ] `pages/project/assessment/`：做题界面，复用现有 `problem-card` 组件，支持单选/填空
- [ ] `pages/project/report/`：

  - 「确认画像，生成计划」CTA 按钮

**前端：新增公共组件**

### M2a · 计划生成
**时间：Week 6–7（2026-06-05 ~ 2026-06-18）**

#### 核心目标
> AI 基于能力画像生成个性化日历计划，家长/学生确认后 home 日历自动填充。

---

#### 技术任务

**后端：Prisma Schema 扩展（M2a 新增/修改）**
```prisma
StudyPlan           // 学习计划总表
Milestone           // 阶段里程碑
TrainingPlan        // 追加字段：planId? / nodeIds[] / milestoneId?
```
- [ ] 运行 `prisma migrate` 生成 M2a 迁移文件

**后端：计划生成规则引擎**（`study-plan/plan-generator.service.ts`）
```
① topologicalSort(knowledgeNodes)  按前置依赖排序
② assignWeights(nodes, weakPoints) 薄弱点权重加成（score↓ → 天数↑）
③ distributeDays(nodes, days×0.8)  每日 2-3 个知识点上限，预留 20% 缓冲
④ createMilestones(nodeDays)       按知识领域分组，2-4 周/个里程碑
⑤ flattenToDailyTasks(milestones)  平铺为每日 TrainingPlan 记录
```
- [ ] 12 个基础计划模板（3 能力层次 × 4 时间周期：1/2/3/6 个月）

**后端：M2a API 实现**
```
POST /projects/:id/generate-plan  触发规则引擎 → 返回 StudyPlan 预览
POST /projects/:id/confirm-plan   确认计划 → 批量写 TrainingPlan → status: executing
```
- [ ] `study-plan.controller.ts` + `study-plan.service.ts`

**前端：M2a 页面**
- [ ] `pages/project/plan-preview/`：
  - `components/milestone-timeline/`：里程碑时间线（横向滚动，显示每阶段知识领域）

- [ ] `pages/home/` 状态 B（executing）：日历中显示系统生成任务的标记（与手动任务区分）

---

#### 验收标准

## 七、风险预案

| 风险 | 触发信号 | 应对 |
|------|--------|------|
| **能力报告质量不达预期** | 内测家长访谈中，超过 50% 表示「不够具体」或「感觉像通用报告」 | M1 完成后暂停推进，集中 1 周迭代报告内容和呈现，再次访谈验证 |
| **付费转化率过低（< 5%）** | M3 结束时付费用户 < 3 个 | 深度访谈 10 个未付费用户，找核心阻力；考虑降低价格或延长免费期 |
| **CAC 持续偏高（> 500 元）** | M4 中期（Week 18）推广数据不理想 | 暂停付费投放，转向纯口碑增长；分析小红书内容哪类转化最好 |
| **用户活跃度低（每日完成率 < 30%）** | M2b 内测数据 | 召开用户共创会，找出执行障碍；简化每日最小任务单元；加强提醒文案 |
| **OCR 准确率问题** | 用户反馈或监控告警 | 已有预案：准确率 < 80% 时转人工辅助标注 |
| **NestJS 迁移延期** | M0 第 2 周仍未完成迁移 | 延续现有 Express 架构继续推进 M0 产品任务，NestJS 迁移推至 M1 空档 |
| **教研合伙人迟迟未落地** | M2 结束仍未找到合适人选 | 不阻塞产品推进；创始人自验为主，合伙人是加分项，不是硬前提 |
| **找到的人不愿 All in** | 对方只愿「顾问」身份，不愿股权绑定 | 直接放弃；利益不对齐的合作比没有更坏 |
